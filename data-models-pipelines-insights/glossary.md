# Glossário Técnico: Data Models, Pipelines and Insights

Este glossário reúne os principais termos e definições abordados no Curso 2 do Google Business Intelligence Professional Certificate, servindo como uma base de consulta rápida para conceitos de engenharia e modelagem de dados.

---

## Glossário do Módulo 3

### Qualidade e Governança de Dados
- **Precisão**: Elemento de teste de qualidade usado para confirmar que os dados estão em conformidade com a entidade real que está sendo medida ou descrita.
- **Regra de negócios**: Uma declaração que cria uma restrição em partes específicas de um banco de dados.
- **Completude**: Elemento de teste de qualidade usado para confirmar que os dados contêm todos os componentes ou medidas desejados.
- **Conformidade**: Elemento de teste de qualidade usado para confirmar que os dados se encaixam no formato de destino exigido.
- **Consistência**: Elemento de teste de qualidade usado para confirmar que os dados são compatíveis e estão de acordo em todos os sistemas.
- **Dicionário de dados**: Uma coleção de informações que descreve o conteúdo, o formato e a estrutura dos objetos de dados em um banco de dados, bem como seus relacionamentos.
- **Linhagem de dados**: O processo de identificação da origem dos dados, para onde eles se deslocaram no sistema e como se transformaram ao longo do tempo.
- **Mapeamento dos dados**: O processo de correspondência de campos de uma fonte de dados para outra.
- **Integridade**: Elemento de teste de qualidade usado para confirmar que os dados são precisos, completos, consistentes e confiáveis em todo o seu ciclo de vida.
- **Teste de qualidade**: O processo de verificação de defeitos nos dados para evitar falhas no sistema; envolve os sete elementos de validação (completude, consistência, conformidade, precisão, redundância, integridade e pontualidade).
- **Redundância**: Elemento de teste de qualidade usado para confirmar que não são movidos, transformados ou armazenados mais dados do que o necessário.
- **Validação de esquema**: Um processo para garantir que o esquema de dados do sistema de origem corresponda ao esquema de dados do banco de dados de destino.
- **Pontualidade**: Um elemento de teste de qualidade usado para confirmar que os dados são atuais.

## Termos e Definições de Módulos Anteriores

### A
- **Interface de Programação de Aplicação (API)**: Um conjunto de funções e procedimentos que integram programas de computadores, formando uma conexão que permite a comunicação entre eles.
- **Desenvolvedor de software de aplicativos**: Uma pessoa que projeta aplicativos de computador ou móveis, geralmente para consumidores.
- **Atributo**: Em um modelo dimensional, uma característica ou qualidade usada para descrever uma dimensão.

### B
- **Business Intelligence (BI)**: Automatização de processos e canais de informação para transformar dados relevantes em insights acionáveis que são facilmente disponibilizados aos tomadores de decisão.
- **Governança de Business Intelligence (BI)**: Um processo para definir e implementar sistemas e estruturas de business intelligence em uma organização.
- **Monitoramento de Business Intelligence (BI)**: Criação e uso de ferramentas de hardware e software para analisar dados com facilidade e rapidez e permitir que as partes interessadas tomem decisões de negócios impactantes.
- **Estágios de Business Intelligence (BI)**: A sequência de estágios que determinam o valor comercial do BI e a maturidade dos dados organizacionais (capturar, analisar e monitorar).
- **Estratégia de Business Intelligence (BI)**: O gerenciamento das pessoas, dos processos e das ferramentas usadas no processo de business intelligence.

### C
- **Banco de dados colunar**: Um banco de dados organizado por colunas em vez de linhas.
- **Sistemas combinados**: Sistemas de banco de dados que armazenam e analisam dados no mesmo local.
- **Linguagem de programação compilada**: Uma linguagem de programação que compila instruções codificadas que são executadas diretamente pela máquina de destino.
- **Contenção**: Quando dois ou mais componentes tentam usar um único recurso de forma conflitante.

### D
- **Analistas de dados**: Pessoas que coletam, transformam e organizam dados.
- **Disponibilidade de dados**: O grau ou a extensão em que as informações oportunas e relevantes estão prontamente acessíveis e podem ser utilizadas.
- **Profissionais de Governança dos dados**: Pessoas que são responsáveis pelo gerenciamento formal dos ativos de dados de uma organização.
- **Integridade dos dados**: A precisão, completude, consistência e confiabilidade dos dados em todo o seu ciclo de vida.
- **Data lake (lago de dados)**: Um sistema de banco de dados que armazena grandes quantidades de dados brutos em seu formato original até que sejam necessários.
- **Data Mart**: Um banco de dados orientado por assunto que pode ser um subconjunto de um Data Warehouse maior.
- **Maturidade dos dados**: O grau em que uma organização é capaz de usar seus dados de forma eficaz para extrair insights acionáveis.
- **Modelo de dados**: Uma ferramenta para organizar os elementos de dados e como eles se relacionam entre si.
- **Particionamento de dados**: O processo de divisão de um banco de dados em partes distintas e lógicas para melhorar o processamento de consultas e aumentar a capacidade de gerenciamento.
- **Pipeline de dados**: Uma série de processos que transporta dados de diferentes fontes para seu destino final de armazenamento e análise.
- **Visibilidade dos dados**: O grau ou a extensão em que as informações podem ser identificadas, monitoradas e integradas a partir de fontes internas e externas diferentes.
- **Data Warehouse**: Um tipo específico de banco de dados que consolida dados de vários sistemas de origem para garantir a consistência, a precisão e o acesso eficiente aos dados.
- **Especialistas em armazenamento de dados**: Pessoas que desenvolvem processos e procedimentos para armazenar e organizar dados de forma eficaz.
- **Migração de banco de dados**: Movimentação de dados de uma plataforma de origem para outro banco de dados de destino.
- **Desempenho do banco de dados**: Uma medida da carga de trabalho que pode ser processada por um banco de dados, bem como os custos associados.
- **Entregável**: Qualquer produto, serviço ou resultado que deve ser alcançado para concluir um projeto.
- **Desenvolvedor**: Uma pessoa que usa linguagens de programação para criar, executar, testar e solucionar problemas de software de aplicativos.
- **Dimensão (Modelagem de dados)**: Uma parte da informação que fornece mais detalhes e contexto sobre um fato.
- **Tabela de dimensões**: A tabela em que são armazenados os atributos das dimensões de um fato.
- **Padrão de design**: Uma solução que usa medidas e fatos relevantes para criar um modelo de apoio às necessidades comerciais.
- **Modelo dimensional**: Um tipo de modelo relacional que foi otimizado para recuperar rapidamente os dados de um Data Warehouse.
- **Banco de dados distribuído**: Uma coleção de sistemas de dados distribuídos em vários locais físicos.

### E
- **ELT (Extrair, carregar e transformar)**: Um tipo de pipeline de dados que permite que os dados sejam coletados de data lakes, carregados em um sistema de destino unificado e transformados em um formato útil.
- **ETL (Extrair, Transformar, Carregar)**: Um tipo de pipeline de dados que permite que os dados sejam coletados de sistemas de origem, convertidos em um formato útil e trazidos para um Data Warehouse ou outro sistema de destino unificado.
- **Aprendizado experimental**: Compreensão por meio da prática.

### F
- **Fato**: Em um modelo dimensional, uma medida ou métrica.
- **Tabela de fatos**: Uma tabela que contém medidas ou métricas relacionadas a um determinado evento.
- **Chave estrangeira**: Um campo em uma tabela de banco de dados que é uma chave primária em outra tabela.
- **Dados fragmentados**: Dados divididos em muitas partes que não são armazenadas juntas, geralmente como resultado do uso frequente dos dados ou da criação, exclusão ou modificação de arquivos.
- **Linguagem de programação funcional**: Uma linguagem de programação modelada em torno de funções.

### G
- **Google DataFlow**: Um serviço de processamento de dados sem servidor que lê dados da origem, transforma-os e grava-os no local de destino.

### I
- **Índice**: Uma tag organizacional usada para localizar rapidamente os dados em um sistema de banco de dados.
- **Profissionais de tecnologia da informação**: Pessoas que testam, instalam, consertam, atualizam e mantêm soluções de hardware e software.
- **Linguagem de programação interpretada**: Uma linguagem de programação que usa um intérprete, normalmente outro programa, para ler e executar instruções codificadas.
- **Iteração**: Repetição de um procedimento várias vezes para se aproximar cada vez mais do resultado desejado.

### K
- **Indicador principal de desempenho (KPI)**: Um valor quantificável, intimamente ligado à estratégia de negócios, que é usado para acompanhar o progresso em direção a uma meta.

### L
- **Modelagem de dados lógicos**: Representação de diferentes tabelas no modelo de dados físicos.

### M
- **Métrica**: Um único ponto de dados quantificável que é usado para avaliar o desempenho.

### O
- **Linguagem de programação orientada a objetos**: Uma linguagem de programação modelada em torno de objetos de dados.
- **Sistema OLAP (Online Analytical Processing)**: Uma ferramenta que foi otimizada para análise, além do processamento, e pode analisar dados de vários bancos de dados.
- **Banco de dados OLTP (Online Transaction Processing)**: Um tipo de banco de dados que foi otimizado para processamento de dados em vez de análise.
- **Otimização**: Maximizar a velocidade e a eficiência com que os dados são recuperados para garantir altos níveis de desempenho do banco de dados.

### P
- **Portfólio**: Uma coleção de materiais que podem ser compartilhados com possíveis empregadores.
- **Chave primária**: Um identificador em um banco de dados que faz referência a uma coluna ou a um grupo de colunas em que cada linha identifica exclusivamente cada registro na tabela.
- **Gerente de projetos**: Uma pessoa que lida com as etapas diárias, o escopo, o cronograma, o orçamento e os recursos de um projeto.
- **Patrocinador do projeto**: Uma pessoa que tem responsabilidade geral por um projeto e estabelece os critérios de sucesso do mesmo.
- **Python**: Uma linguagem de programação de uso geral.

### Q
- **Plano de consulta**: Uma descrição das etapas de um sistema de banco de dados para executar uma consulta.

### R
- **Recursos**: As ferramentas de hardware e software disponíveis para uso em um sistema de banco de dados.
- **Tempo de resposta**: O tempo que um banco de dados leva para concluir uma solicitação do usuário.
- **Banco de dados baseado em linhas**: Um banco de dados que é organizado por linhas.

### S
- **Sistemas de armazenamento e computação separados**: Bancos de dados em que os dados são armazenados remotamente e os dados relevantes são armazenados localmente para análise.
- **Banco de dados com hospedagem única**: Banco de dados em que todos os dados são armazenados no mesmo local físico.
- **Esquema Snowflake**: Uma extensão de um esquema em estrela com dimensões adicionais e, muitas vezes, subdimensões.
- **Esquema em estrela**: Um esquema que consiste em uma tabela de fatos que faz referência a qualquer número de tabelas de dimensões.
- **Estratégia**: Um plano para atingir uma meta ou chegar a um estado futuro desejado.
- **Orientado por assunto**: Associado a áreas ou departamentos específicos de uma empresa.
- **Analista de sistemas**: Uma pessoa que identifica maneiras de projetar, implementar e aprimorar sistemas de informação para garantir que eles ajudem a possibilitar a realização dos objetivos de negócios.
- **Desenvolvedor de software de sistemas**: Pessoa que desenvolve aplicativos e programas para os sistemas de processamento de back-end usados nas organizações.

### T
- **Tática**: Um método usado para possibilitar uma realização.
- **Tabela de destino**: O local predeterminado para o qual os dados do pipeline são enviados a fim de serem processados.
- **Taxa de transferência**: A capacidade geral do hardware e do software do banco de dados para processar solicitações.
- **Habilidade transferível**: Uma capacidade ou proficiência que pode ser aplicada de um trabalho para outro.

### V
- **Métrica de vaidade**: Pontos de dados que têm a intenção de impressionar os outros, mas não são indicativos do desempenho real e, portanto, não podem revelar nenhuma percepção comercial significativa.

### W
- **Carga de trabalho**: A combinação de transações, consultas, análises de data warehousing e comandos do sistema que estão sendo processados pelo sistema de banco de dados em um determinado momento.
